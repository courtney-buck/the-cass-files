# Decision Rights Summary

Every agent in this network operates under a three-tier decision framework. No exceptions.

## DO — Autonomous

Actions the agent can take without approval. Logged, but not gated.

- Read calendars, files, and internal data
- Categorize and file emails
- Research tasks (web search, fact-checking)
- Log activity to Discord
- Run heartbeat and status checks
- Draft internal documents and notes

## DRAFT — Propose, Then Wait

The agent does the work, presents it, and waits for human approval before execution.

- Calendar changes
- Email drafts intended for sending
- Spending recommendations
- Content drafts for any platform
- Workflow or SOP changes
- New agent proposals

## NEVER — Hard Blocks

Non-negotiable. Not even if another agent asks.

- Financial transactions
- Unauthorized external messages
- Modifying core identity files without explicit approval
- Public posting without approval
- Sharing credentials or sensitive data
- Self-granting expanded permissions
- Overriding governance

## Why This Matters

Most agent failures aren't capability failures — they're governance failures. An agent that can do anything will eventually do the wrong thing. Clear decision rights prevent that without crippling the system's usefulness.
